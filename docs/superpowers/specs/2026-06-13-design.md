# 変更仕様書: group() のキー再出現をエラーにする

- 作成日: 2026-06-13
- 起点: docs/design/design-concept.md(To Do「group のキー再出現をエラーにする」2026-06-11・Spool 設計合意より)

## 1. 目的

`group(@groups)` は入力がソート済みであることを前提とする。現在の実装(`_group_rows`)は各キー組をハッシュバケットでまとめるため、**一度終わったキー組が離れて再出現しても黙って同一 node にマージ**してしまう。これを「呼び出し側のソート漏れ(SQL の `ORDER BY` 忘れ等)」として検出し `croak` する。

## 2. 検証規則

各階層の「外側からの累積キー組」について、同一のキー組は**連続して現れなければならない**。

- 累積キー組 = `level1` のカラム群、`level1+level2` のカラム群、… と、外側から内側へ向かって各レベルまでのグループカラムを連結したもの。
- 入力 `AOH` を順に走査し、各レベル L の累積キー組タプルについて「一度出現が途切れたタプルが再び現れた」ら `croak` する。
- 全レベル(`level1` 〜 最深 `levelN`)について確認する。最深の全結合キーだけでは、上位レベルの再出現を捕捉できないため(例: `(A,x),(B,y),(A,z)` は全結合キーは各1回だが `level1=A` が分断される)、各レベルを個別に確認する。

## 3. 挙動

- 違反時: `croak`(メッセージに再出現したキー組・レベルが分かる情報を含める。例: `group: key reappears (unsorted input): ...`)。
- 連続した(ソート済みの)入力に対する結果は**従来と完全に同一**。`group` / `expand` の可逆性は維持される。
- 葉(`@rest`)の順序・内容は変更しない。
- 既存の検証(`grouped` 状態・空 groups・未知キー・グループ内/間のキー重複)は従来どおり先に行う。

## 4. 実装方針

- `group()` 内、`_group_rows` 呼び出しの前に、ソート済み前提の検証を1パスで行う関数(例: `_check_group_sorted($rows, \@groups)`)を追加する。
- 各レベル L について累積カラム列を作り、行を順に走査して「直前のタプルと異なるのに過去に出現済み」のタプルを検出したら `croak`。
- タプル化は `_group_rows` と同じ区切り(`\x1E` と長さ前置)でキー衝突を避ける。

## 5. テスト方針(test/metaaoh.t)

| 区分 | ケース |
| --- | --- |
| 回帰 | ソート済み入力の `group` は従来どおりの `AOT` を返す。`group`→`expand` が元に戻る |
| 単一レベル再出現 | `level1` キーが分断して再出現する入力(例 `A,B,A`)で `croak` |
| 多レベル再出現 | 内側レベルの累積キー組が分断して再出現する入力で `croak` |
| 全結合キーでは検出不能なケース | `(A,x),(B,y),(A,z)` のように最深キーは各1回でも `level1` が分断する入力で `croak` |
| 境界 | 連続する重複キー(正常)・1行・全行同一キーは `croak` しない |
