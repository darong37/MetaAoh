コードレビューで発見された10件のバグ・問題をすべて修正する。

## Bug 1 — `_expand_rows` の "expand missing key" が発火しない（line 276–279）

### 問題
`map { $_ => $merged{$_} } @$cols` でハッシュを作ると、値が `undef` でもキーが作られる。
`exists $flat{$key}` は常に `1` になるため "expand missing key" croak は到達不能。
カラム欠如時も "expand undef value" という誤ったメッセージが出る。

### 修正
`%flat` を作る前に `%merged` を直接確認する順序に変える。

```perl
for my $key (@$cols) {
    croak "expand missing key: $key" unless exists $merged{$key};
    croak "expand undef value: $key" unless defined $merged{$key};
}
my %flat;
@flat{@$cols} = @merged{@$cols};
```

## Bug 2 — `new` が呼び出し元配列をそのまま bless する（line 39）

### 問題
`bless $aoh, $cls` は渡された配列リファレンスをそのまま使うため、
呼び出し元の変数がサイレントに MetaAoh オブジェクトになる。

### 修正
bless 前にシャローコピーを取る。

```perl
my $obj = [@$aoh];
bless $obj, $cls;
$state_of{ refaddr($obj) } = { meta => { ... } };
return $obj;
```

## Bug 3 — `sort` が呼び出し元の配列を書き換える（line 114）

`new` でコピーを取れば `@$self = @rows` はオブジェクト内部にのみ影響する。
`sort` 本体は変更しない。

## Bug 4 — `expand` の返り値が flat/grouped で非対称（line 169）

### 問題
grouped → 新しいオブジェクトを返す
flat → `$self` をそのまま返す（同じ参照）

`$copy = $m->expand; $copy->add(...)` が flat のとき `$m` も変更されてしまう。

### 修正
flat のときも新しいオブジェクトを返す。

```perl
sub expand {
    my ($self) = @_;
    my $aoh = $self->meta->{grouped}
        ? _expand_rows($self, $self->meta->{cols}, {})
        : $self->toAoh;
    return ref($self)->new($aoh, @{ $self->meta->{order} });
}
```

注：`validate` の二重呼び出しも同時に解消される（`new` 内で1回行われるため十分）。

## Bug 5 — `\x1E` バケットキー衝突（line 232）

### 問題
グループキーに `\x1E` を区切り文字として使うと、カラム値に `\x1E` が含まれる場合に
異なる行が同一バケットに誤って分類される。

### 修正
各カラム値の長さをプレフィックスとして付けることで、値の境界を明確にする。

```perl
my $bucket_key = join "\x1E", map {
    my $v = defined $row->{$_} ? $row->{$_} : '';
    length($v) . ":" . $v;
} @$cols;
```

これにより `"a\x1Eb"` と `"a"` + `"b"` は区別される。

## Bug 6 — カラム名正規表現がアンダースコアを禁止（line 25）

### 問題
`^([A-Za-z][A-Za-z0-9]*)` は `_id`、`col_name` など実用的なカラム名を拒否する。
`design-concept.md` にそのような制限の記述はない。

### 修正
アンダースコアと先頭アンダースコアを許可する。`\x1E`（バケットキー区切り）と `#`（属性指定）と制御文字のみ禁止。

```perl
if ($spec !~ /^([^\x00-\x1F#\x1E]+)(#?)$/) {
    croak "bad order: $spec";
}
```

## Bug 7 — `DESTROY` での refaddr 再利用リスク（line 179）

### 問題
`%state_of` はメモリアドレス（refaddr）をキーにする inside-out オブジェクト方式。
Perl がアドレスを再利用した場合、DESTROY のタイミングによって別オブジェクトのメタを消す可能性がある。

### 修正
`Hash::Util::FieldHash` を使う。これは弱参照ベースで自動的にエントリを管理し、
refaddr 再利用問題と DESTROY 手書きの両方を不要にする。

```perl
use Hash::Util::FieldHash qw(fieldhash);
fieldhash my %state_of;
# DESTROY サブルーチンは削除可
```

`Hash::Util::FieldHash` はコアモジュール（Perl 5.10+）なので追加依存なし。

## Bug 8 — sort テストデータが数値/文字列判定バグを検出できない（test line 52）

### 問題
`'05'`、`'10'`、`'20'` は数値順・辞書順どちらでも同じ順序になるため、
num/str ディスパッチが壊れていてもテストが通る。

### 修正
`'9'` を追加する（数値: 9 < 10、辞書: '9' > '10'）。

```perl
{ name => 'a', age => '9' },   # 追加
```

期待出力も合わせて更新する。

## Bug 9 — `delete $merged{'*'}` がデッドコード（line 263）

### 問題
`$base` に `'*'` が含まれることは構造上ありえない。
初回は `{}` を渡し、再帰呼び出しは `'*'` を除外して構築した `%merged` を渡す。

### 修正
該当行を削除する。

## Bug 10 — `validate` 内の2番目ループが冗長（line 209）

### 問題
カラム数チェック（`@keys == $expected`）と全必須キー存在チェックが通った時点で、
余分なキーは論理的に存在できない。2番目の `for my $key (@keys)` ループは到達不能。

### 修正
2番目のループを削除する。

```perl
# 削除する
for my $key (@keys) {
    croak "unknown key: $key" unless $required{$key};
}
```
